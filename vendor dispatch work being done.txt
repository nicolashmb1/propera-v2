Propera HVAC Vendor Dispatch System Map

Version: v1
Purpose: Automatically dispatch HVAC vendors when a maintenance ticket is created.

1. High-Level Flow
Tenant SMS
   ↓
Router (handleSmsRouter_)
   ↓
Core Pipeline
   ↓
Ticket Created
   ↓
WorkItem Created
   ↓
PolicyEngine (WORKITEM_CREATED event)
   ↓
PolicyRule Match (HVAC)
   ↓
ASSIGN_VENDOR_BY_POLICY action
   ↓
Ticket Assigned to Vendor
   ↓
assignTicketFromRow_()
   ↓
notifyVendorForTicket_()
   ↓
Vendor receives SMS
   ↓
Vendor replies YES / NO
   ↓
Vendor command handler
   ↓
Ticket + WorkItem updated

6. Policy Action

New PolicyEngine action

ASSIGN_VENDOR_BY_POLICY

Purpose

Assign vendor automatically based on PropertyPolicy

Steps executed

vendorId = ppGet_(prop,"HVAC_VENDOR_ID")

Ticket row updated

AssignedType = "Vendor"
AssignedId   = vendorId

Then dispatch triggered

assignTicketFromRow_()

10. Safety Controls
Dry Run

PropertyPolicy flag

POLICY_ENGINE_DRY_RUN = TRUE

Effects

Rules match
Actions logged
NO vendor assignment
NO SMS sent
Idempotency / Spam Protection

Dispatch prevented when

AssignedType = Vendor
AssignedId = same vendor
VendorStatus not empty

Log

POLICY_VENDOR_ALREADY_ASSIGNED
